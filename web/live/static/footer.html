About Livelogs

Livelogs is written with the intention of providing easily accessible logs for users of ozfortress servers, as well as providing server owners the opportunity to have live, web-based, logs with the ease of a simple plugin installation.

Livelogs uses a SourceMod plugin to handle Source server-side data and information management, such as initiating the connection with, and sending the appropriate identification data to, the external Livelogs daemon, running on a server provided by ozfortress and hosted by infinite networks. The plugin also provides a SourceTV2D server (if enabled), by which users may view a 2D SourceTV broadcast (delayed by tv_delay) of the server the plugin is running on.

The plugin does not send any sensitive information (such as passwords), it simply sends a packet with information containing the server's ip, port, name (configurable), and if applicable, the SourceTV2D port (configurable). The plugin source code will be made available for users wishing to modify it, or confirm for themselves that no sensitive data is being sent.

All log data is sent via the Source engine's built in UDP logging functionality.

As Livelogs is still being developed to an extent, you may notice some debug information during use of the website as well as missing maps and green filler for SourceTV2D.
